# Claude Code 作業ルール

## ドキュメント更新ルール

以下の変更を行った場合は、必ず `docs/DEPLOYMENT.md` を更新すること：

1. **プロジェクト構成の変更**
   - ファイル/フォルダの追加・削除・名前変更
   - 新しい依存関係の追加

2. **URL/エンドポイントの変更**
   - Vercel URLの変更
   - API URLの変更
   - LIFF設定の変更

3. **環境変数の変更**
   - 新しい環境変数の追加
   - 既存の環境変数の変更

4. **連携システムの変更**
   - potz側のファイル変更
   - LINE設定の変更

5. **処理フローの変更**
   - 機能追加・削除
   - 動作の変更

## コミット時の注意

- 大きな仕様変更時はコミットメッセージに変更内容を明記
- ドキュメント更新も同じコミットに含める

## プロジェクト概要

高齢親の安否確認アプリ（気分選択→音声応答→LINE通知）

- **フロント**: 静的HTML（Vercel）
- **バックエンド**: potz Laravel（AWS）
- **通知**: LINE Messaging API
